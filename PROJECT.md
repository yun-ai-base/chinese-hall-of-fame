# 太阳系中华名人堂 · Celestial Mandate（chinese-hall-of-fame）

以太阳系为隐喻框架，将中华五千年文明可视化为可探索的交互星系，采用**严格四层下钻**结构：太阳=中华文明（L1）→ 八大行星=八大文化维度（L2）→ 分类星球=各维度下的子分类（L3，每个维度一屏只展示其分类星球，不混入名人）→ 名人卫星=该分类下的核心人物（L4，共 361 位）。逐层点击下钻，上层轨道平滑淡出作背景，避免视觉干扰。

## 技术栈
- 纯静态前端：HTML5 + CSS3 + ES Modules（无构建工具）
- 3D 渲染：Three.js r170（vendor 本地加载，importmap 指向相对路径 `./vendor/three/`）
- 数据驱动：所有展示内容来自 JSON，渲染层不硬编码任何名人信息
- 字体：Noto Serif/Sans SC（Google Fonts CDN）

## 运行方式
> 必须用 HTTP 服务器（ES Modules + fetch 不支持 `file://`）：
```bash
cd chinese-hall-of-fame
python -m http.server 8000
# 浏览器打开 http://localhost:8000
```

## 已实现功能（对应设计文档 Phase 1~5）
- **严格四层星系结构（下钻式）**：
  - L1 **太阳** = 中华名人堂总览（球体中心嵌「华夏」二字，尺寸较小、暗金细底、白字细黑边，避免成为唯一焦点；持续自转）
  - L2 **八大文化维度行星**（儒释道、书画、文学、科技、政治、军事、史学、医药…），点击进入该维度
  - L3 **分类星球** = 进入维度后**只展示该维度的分类**（如「书画艺术」仅展示 书法/人物花鸟/山水画/明清文人/近现代/篆刻 6 个分类星球），**不混入名人**；点击某分类星球（如「书法」）下钻
  - L4 **名人卫星** = 该分类下的核心人物（如书法 11 位），环绕**中央分类恒星**按年代分圈公转；每个轨道**最多 2 颗名人**（按出生年份分轨），公转速度整体减慢；点击名人展开信息面板
  - 每一层级都有一颗**持续自转的中央恒星**：L1 太阳（中华名人堂，中心嵌「华夏」，自转速度 `0.004` rad/帧）→ L2 维度恒星（维度名，位于分类层中心）→ L3 分类恒星（分类名，位于名人层中心，且固定可见，修复「点击分类后中心不可见」）
  - 分类星球（L3）与名人卫星（L4）均带**独立轨道环轨迹**、**各自公转**（开普勒式内快外慢）、**错位同心轨道**（按数量分 1~2 圈、不同半径与相位），并各自**自转**
  - 逐层下钻时**上层轨道平滑淡出**作背景（L3 分类星球随 L4 名人视图淡出为背景），返回时自动恢复亮度
- **星空氛围与行星特征**：
  - **中国星官图**（`ChineseStarMap` 替代原随机 `StarField`）：深空以程序化坐标绘制简化的中式星象——**北斗七星**（勺形连线）、**二十八宿**（四象各 7 宿弧线：东青龙/北玄武/西白虎/南朱雀）、**三垣**（紫微/太微/天市 3 小弧），外加大量稀疏淡尘星避免虚空；星点为淡金/青白 Points + 逐星微闪 ShaderMaterial，星官连线为淡金 LineSegments + 轻微呼吸式透明度，强化文化星空质感
  - **轨道层级区分**（`OrbitRing` 改用 `Line2`/`LineMaterial` 真实像素线宽）：① L1 宇宙层轨道**实线**、按轨道半径做「内层细（1.5px）外层粗（3.3px）」；② L2/L3/L4 深层轨道**虚线** + 较细线宽（1.3~1.6px）；③ 全部轨道用 **`OrbitRing.desat()` 降饱和维度色**（避免深色背景下高饱和原色糊成一片）；④ **hover 高亮**：鼠标悬停某星球，其 `userData.orbitRing` 经 `Raycaster` 的 `hover` 事件调用 `setHighlight(true)`——环体提亮暖白 + 线宽加粗（×2.4）+ 透明度抬升至 0.9，移开复原。线宽依赖 `LineMaterial.resolution`，由 `SceneManager` 创建前与 resize 时统一刷新（`OrbitRing` 维护实例表）
  - **印章式品牌装饰**（标题区）：`index.html` 的 `#title-display` 增加朱红圆角方印（2×2「中華名人」白文、轻微旋转 -4° + 辉光），强化「名人堂」文化质感；字体仍用思源宋体（用户明确选择「仅加印章不改字体」，不引入书法体）
  - **行星环（多类型）**（`Planet._createRing(type)` + `OrbitSystem.FEATURES`）：按真实太阳系映射为四颗行星位分配不同环——**土星位**（idx6=书画艺术）明显宽环（细密环纹 + 卡西尼缝 + 约 24° 倾角，米金色）；**木星位**（idx5=科技工程）极淡单薄尘环；**天王星位**（idx7=政治治世）黯淡窄环、**近乎竖直**（还原天王星 ~98° 自转轴倾角）；**海王星位**（idx8=探索交流）微弱环带 + **角向弧段**（还原 Adams 环弧团块）。统一程序化 ShaderMaterial（`uBands/uCassini/uArcs/uAlpha`），随行星公转、随下钻 `fade` 联动淡出
  - **木星大红斑**（`Planet._createRedSpot`）：木星位（科技工程）表面赤道偏南嵌一枚程序化红棕椭圆贴片，作为球体子节点随本轴自转横扫、转到背面自动被球体深度遮挡隐没
  - **地球月亮**（`Planet._createMoon`）：地球位（诗词文学）带一颗灰色月亮绕地公转（`moonGroup` 挂在公转 group 上、自身独立旋转）；**进入 L2 及更深层时自动隐藏**（`setMoonVisible(!dimId)`）避免视觉干扰，返回宇宙层恢复
  - **金星逆向自转**（`Planet.retrograde`）：金星位（哲学思想）本轴自转方向取反，还原金星逆行自转的真实特征
  - **轨道拖尾**（`Planet._createTrail`）：维度行星沿公转反方向拖出同色渐隐彗星尾，强化运动方向感；弧长 0.9 弧度（约 51.6°）、头部透明度 0.7、幂次衰减使头部聚拢长尾渐隐，加色混合叠在星海上
  - URL hash 持久化升级：`#/u` · `#/d/:dimId` · `#/c/:dimId/:category`（新增分类层） · `#/f/:figureId`
- **配色层级区分（colorScale）**：每个子分类在「维度主色」锚定的色环上均匀铺开（`categoryPalette`），彼此区分度最大；某分类下的名人卫星在其「分类主色」附近均匀铺开（`figureColor`），彼此区分且与中央分类恒星明显不同；L3 分类星球颜色 == 下钻后 L4 中央恒星颜色（保证下钻连续性）。探索交流维度基础色由青绿 `#1ABC9C` 调整为青蓝 `#0FB5C9`，与神话传说绿明显区分。
- **名人选中（L4 第四级结构）**：在分类名人层点击某颗名人卫星 → 聚焦 + 该卫星放大高亮（其余保持常态）+ 顶部出现**四级头部文字**（中华名人堂 › 维度 › 分类 › 名人）+ **背景极淡染色**（取该名人主色相），构成明确可感知的第四级；点击空白或返回即取消选中、还原背景。
- **行星名内嵌星球**：维度名写在星球圆盘内部（白字深色描边 + `depthTest:false` + 高 `renderOrder`），不漂浮在外；**名人卫星名字同样内嵌于球体内部**（原为漂浮在外侧的标签），随星球大小自适应缩放。**中央恒星（维度/分类）名字采用「干净标签」**：同色系半透明圆角底 + 白字 + 细黑边，无厚重描边/发光，避免成为中心球时产生两层重影。
- **下钻时上层淡出为背景**：
  - 宇宙 → 维度：8 条宇宙轨道环整体淡出（当前维度轨道保留作锚点），非当前维度行星变暗，当前行星保持明亮但**隐藏其内嵌标签**，避免与 L2 维度恒星标签重叠
  - 维度 → 分类/名人：维度层（L2 中心恒星 + 分类轨道环 + 分类星球）淡出为背景；作为父视图时，**中心恒星完全隐藏**，轨道环与分类星球同步变淡，避免与当前层中心球/文字重叠
  - 分类/名人视图下：当前维度行星与轨道环作为锚点进一步调暗（行星 0.25 / 轨道环 0.12），避免与当前层中心球抢视觉焦点
  - 返回时上层自动恢复亮度

## 宇宙视图布局说明（太阳系隐喻）
- 太阳本体半径 5，光晕 sprite scale 18，保持居中且不过度遮挡内侧轨道。
- 八大行星轨道半径按太阳系式非线性拉开：`[15, 21, 28, 36, 46, 58, 71, 85]`（内紧外疏）。
- 行星大小分级：内圈岩质较小（2.0–3.2），外圈气态巨星较大（3.6–5.5）。
- 公转速度遵循开普勒第三定律 `ω ∝ r^-1.5`：内圈快（约 21 秒/圈），外圈慢（约 5 分钟/圈），肉眼可见差距。
- 相机初始位置 `(0, 55, 110)`，`focusUniverse` 回拢也使用同一远距偏移，避免相机贴到太阳上导致白屏/遮挡。
- 轨道环透明度 `0.42` + `depthWrite: false`，在星空背景下清晰可见；下钻到维度时淡出到 `0.05`（当前维度轨道保留为 `0.3` 锚点）。
- 非当前维度行星下钻时变暗到 `0.12`，避免与当前维度争夺视觉焦点。
- **行星渲染与立体感（程序化起伏纹理）**：抛弃纯色球，改用 `utils/planetTexture.js` 生成**多层噪声球面纹理**（mulberry32 确定性种子 → 14 大团 / 38 中斑 / 96 细斑，叠加极区提亮），让球体表面有真实起伏；`MeshStandardMaterial` 以该纹理作 `map` + `emissiveMap`（emissiveMap 为压暗副本，保留背面起伏而不提亮），`roughness=0.55 / metalness=0.08 / emissiveIntensity=0.28`；主明暗交给太阳 `PointLight`（`decay=0` 全轨道覆盖），呈现朝太阳亮、背太阳暗的球体体积；`AmbientLight=0.18` 仅保留背光面基础可见度，不抹平明暗。分类星球（L3）沿用同一套立体渲染。
